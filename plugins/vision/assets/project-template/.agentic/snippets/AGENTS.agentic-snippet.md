## Agentic delivery

Use `$vision` for calibrated pre-goal discovery, bounded implementation, risk-selected verification, diagnosis, and delivery. Read `.agentic/work-environment.md` when present; reuse explicit owner decisions, verify discoverable facts, and resolve its material research queue on a capable machine without treating the profile as approval or evidence. For new contracted work, research first, use bounded read-only scouts only for independent unknowns, synthesize and resolve material choices, freeze the contract, then create the persistent goal using the available authorized mechanism. Resume existing goals/contracts without duplicating them. Use Beads for durable work state, dependencies, decisions, and verifier evidence references; Beads is not the verification authority.

The machine contract is `.agentic/tasks/<bead-id>.json`. New contracts use schema 3 with ready `intake`, a complete `goal_spec`, `risk_gate_version: 1`, and every direct risk mapped to required checks. Existing schema-2 contracts remain compatible. Task size controls planning persistence only; direct risk flags select gates. Generate the exact goal objective with `node .agentic/bin/agentic-harness.mjs goal-spec --task <bead-id>`, then activate its emitted intent with `node .agentic/bin/agentic-lifecycle.mjs activate --task <bead-id> --goal-intent <sha256> --bead <bead-id>`. Use `resume --json` for one next slice. Derived lifecycle and worktree files are not authority. Run `node .agentic/bin/agentic-harness.mjs status --task <bead-id>` before reporting completion. State the honest level: implemented-not-verified, locally-verified, closure-verified, or delivered-and-verified.

Any criterion that depends on first-party service behavior requires a real first-party check. Material visual changes require exact-image structured review. Required advisory lanes remain current-attempt builder evidence and fail when missing, stale, tampered, or inconclusive. A protected verifier/CI lane - not the builder, reviewer, hook, lifecycle cache, or Bead state - issues closure-grade evidence. Delivered-and-verified additionally requires a distinct signed delivery-controller attestation.
